# ADR-004: Чистота Router-узлов в LangGraph

**Статус:** ACCEPTED (подтверждено в Beta-0.1a)  
**Дата:** 2026-06-16

## Контекст
Смешивание логики маршрутизации и мутации состояния в одном узле приводит к сложным для отладки ошибкам.

## Решение
Router-узлы должны быть чистыми функциями (возвращают только строку — имя следующего узла). Вся мутация состояния должна происходить строго в узлах обработки.

## Подтверждение
В Beta-0.1a функция `router_node` возвращает только строку (`"coder_node"` или `"summary_node"`) на основе `state["file"]["status"]`. Мутация состояния происходит в `coder_node`, `write_and_validate_node` и `summary_node`. Цикл маршрутизации подтверждён логами:
- `evidence/2026-06-16_1425_beta01a_run.log`
- `evidence/2026-06-16_1800_beta01a_repro_run.log`

## Последствия
Предсказуемое поведение графа, упрощённая отладка и возможность независимого тестирования логики переходов.
