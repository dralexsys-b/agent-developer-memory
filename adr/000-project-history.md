# ADR-000: История проекта и начальная реконструкция

**Статус:** ACCEPTED  
**Дата:** 2024-06-16

## Контекст
Проект начался с задачи создания системы для автономной разработки и генерации кода.

## Решение (Историческая реконструкция)
В ходе проектирования были пройдены следующие этапы:
1. Обсуждение и отказ от интерактивных TUI-подходов в пользу детерминированных систем.
2. Выбор LangGraph как основы для оркестрации.
3. Разработка и согласование Lean Project Memory Architecture v1.0 для управления контекстом.
4. Согласование ключевых архитектурных паттернов как `AGREED_DESIGN`:
   - FileState FSM v1.3
   - Knowledge Supremacy Matrix
   - Pure Router Rules

## Последствия
Этот документ фиксирует отправную точку проектных решений. Конкретные инфраструктурные факты (наличие моделей, состояние файлов) вынесены в `INFRASTRUCTURE.md` и `evidence/`, а не хранятся здесь.
